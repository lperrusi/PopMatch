# PopMatch Screenshot Shot-List (Portfolio Safe)

Use this sequence for a clean story arc: problem setup -> discovery experience -> decision support -> retention loop -> product maturity.

## Capture Standards
- Use consistent device frame, resolution, and status bar style.
- Prefer realistic but anonymized test account data.
- Keep one visual focal point per screenshot.
- Export all images at the same aspect ratio for portfolio consistency.

## Recommended Sequence (12 Shots)

### 1) Onboarding: Preference Setup
- **Purpose:** Show how the product captures intent early.
- **Composition Notes:** Include genre/platform selection controls.
- **Caption:** "Onboarding captures user taste signals early to reduce cold-start recommendation noise."

### 2) Onboarding: First Personalized Transition
- **Purpose:** Show immediate perceived value after setup.
- **Composition Notes:** Show post-onboarding handoff into discovery.
- **Caption:** "The flow transitions directly from setup to actionable discovery, minimizing drop-off between intent and value."

### 3) Discover Screen: Card Stack (Movies)
- **Purpose:** Highlight primary interaction model.
- **Composition Notes:** One card in focus, next cards visible behind.
- **Caption:** "Swipe-first discovery helps users evaluate content quickly without navigating dense list views."

### 4) Discover Screen: TV Shows Tab
- **Purpose:** Demonstrate cross-content consistency.
- **Composition Notes:** Mirror composition of movie shot for side-by-side comparison.
- **Caption:** "The same interaction pattern scales across movies and shows for lower cognitive load."

### 5) Filter Panel Open (Mood/Genre/Platform)
- **Purpose:** Show user-controlled relevance tuning.
- **Composition Notes:** Capture active filters or selected states.
- **Caption:** "Users can tune recommendations in real time by mood, genre, and platform availability."

### 6) Match Success / Positive Feedback Moment
- **Purpose:** Show delight and behavioral reinforcement.
- **Composition Notes:** Modal/overlay visible with clear next actions.
- **Caption:** "Positive feedback moments reinforce engagement and guide users to next high-intent actions."

### 7) For You / Personalized Recommendations
- **Purpose:** Prove recommendation depth beyond swiping.
- **Composition Notes:** Include varied titles to imply diversity.
- **Caption:** "A personalized feed balances relevance and variety to prevent repetitive recommendation loops."

### 8) Movie Detail Screen
- **Purpose:** Show decision-support depth.
- **Composition Notes:** Include title metadata + watch actions.
- **Caption:** "Detail screens provide enough context to turn interest into confident watch decisions."

### 9) Show Detail: Seasons/Episodes
- **Purpose:** Show long-form content support.
- **Composition Notes:** Show episodes/season structure and progress cues if visible.
- **Caption:** "TV detail supports episodic engagement, enabling continuity and return behavior."

### 10) Watchlist Screen
- **Purpose:** Show retention and intent preservation.
- **Composition Notes:** Include movies and/or shows saved state.
- **Caption:** "Watchlist captures future intent so discovery sessions continue across days."

### 11) Favorites Screen (With Sort Controls)
- **Purpose:** Show user curation and account value growth.
- **Composition Notes:** Include sorting UI or grouped state.
- **Caption:** "Favorites evolve into a personal media library with sorting for quick retrieval."

### 12) Profile + Stats + Settings
- **Purpose:** Show product completeness and account layer.
- **Composition Notes:** Include user stats and settings links.
- **Caption:** "Profile surfaces usage insights and preference controls that continuously improve recommendation quality."

## Optional Bonus Collage (Single Slide)
- Loading state
- Empty state
- No-results state
- Undo feedback state

**Caption:** "Edge-state design ensures the product remains understandable and usable across imperfect data conditions."

## Do-Not-Show Checklist (Protect Your IP)
- Exact algorithm formulas, scoring weights, factor dimensions.
- Internal service/class names and architecture diagrams.
- Experiment/variant configuration specifics.
- Debug screens, internal logs, or model state panels.
- Proprietary roadmap features not yet shipped.

## Safe-To-Say Alternatives
- Say: "Hybrid personalization approach."  
  Avoid: "Weighted blend with exact component percentages."

- Say: "Real-time adaptation from user behavior."  
  Avoid: "Specific learning cadence, thresholds, and update rules."

- Say: "Experiment-ready recommendation architecture."  
  Avoid: "Detailed variant split logic and test definitions."

- Say: "Diversity and freshness balancing."  
  Avoid: "Concrete ranking and penalty equations."
